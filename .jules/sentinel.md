## 2024-04-27 - Fix missing PAM authentication for hyprlock
**Vulnerability:** The NixOS configuration explicitly defined PAM authentication for `swaylock` twice, but omitted it for `hyprlock` which is actually used as the primary screen locker via `hypridle`. When PAM is not explicitly configured for a locker in NixOS, authentication fails closed (lockout) or fails open depending on the system, which represents a critical security risk for physical access.
**Learning:** In NixOS, screen locker binaries are not automatically granted PAM authentication rights. They require an explicit declaration (`security.pam.services.<locker> = {};`). If a configuration migrates to a new locker or uses multiple, all must be explicitly defined. Duplicate entries are often a copy-paste error hiding a missing entry.
**Prevention:** When introducing a new locker like `hyprlock`, always ensure the corresponding `security.pam.services.hyprlock = {};` entry is present in the same module. Avoid copy-pasting existing config lines without adapting them to the new tool.
