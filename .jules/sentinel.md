## 2024-04-15 - [CRITICAL] Fix screen locker PAM authentication bypass/lockout
**Vulnerability:** The NixOS configuration explicitly allowed `swaylock` to unlock the screen via PAM (`security.pam.services.swaylock = {};`), but the actual locker being invoked by `hypridle` was `hyprlock`. This mismatch in configuration could lead to a scenario where the screen locker cannot authenticate users, potentially resulting in user lockouts or forcing users to fail-open the lock screen, bypassing security.
**Learning:** Screen lockers on Wayland (like `swaylock` or `hyprlock`) require explicit PAM service configurations to securely read user authentication tokens. Changing the active locker component without updating its corresponding PAM entry creates critical authentication gaps.
**Prevention:** Always pair changes in the active security component (like screen lockers or display managers) with updates to their required security bindings, specifically PAM configurations in NixOS. Ensure no obsolete configurations are left behind that might lead to unexpected behavior.
