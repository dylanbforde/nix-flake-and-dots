## 2024-10-24 - [CRITICAL] Docker Group Privilege Escalation in NixOS
**Vulnerability:** The user `dylan` was assigned to the `docker` group in `modules/core/users.nix`, despite `virtualisation.docker.rootless.enable = true` being configured. Being in the `docker` group grants equivalent-to-root privileges, as it allows arbitrary container execution which can be used to mount the host filesystem and escalate privileges to root without a password.
**Learning:** In NixOS (and Linux in general), the `docker` group allows unrestricted root access. When rootless Docker is specifically enabled (`virtualisation.docker.rootless.enable = true`), adding users to the `docker` group defeats the purpose of the security isolation provided by rootless Docker, introducing a critical privilege escalation vector.
**Prevention:** Never add users to the `docker` group when using rootless Docker. Ensure configuration reviews verify that the `docker` group is not present in `extraGroups` when rootless options are toggled on.
