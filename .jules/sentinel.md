## 2024-05-24 - Implicit Defaults in Security Boundaries
**Vulnerability:** Core security mechanisms like the firewall and Polkit are left to implicit OS/framework defaults instead of being explicitly enabled in the configuration files (e.g., `networking.firewall.enable = true;`, `security.polkit.enable = true;`). Relying on defaults is risky as these may change upstream or get inadvertently overridden, leading to unintended and potentially insecure configurations (like exposing an open SSH port without a firewall or requiring unsafe passwordless sudo for graphical tasks).
**Learning:** Security configurations should explicitly articulate the required security posture rather than assuming it based on current defaults. This is a critical defense-in-depth practice.
**Prevention:** Always explicitly define required security controls (firewalls, authentication mechanisms, etc.) in configuration files to guarantee the intended security posture, regardless of upstream defaults.
